Account → Partner / Marketing subscription — suggested flow (wireframe)

Idea: There are two account types (Customer + Partner), but Company is a shared billing anchor for subscriptions. Public listing is separate from “has Company” (draft vs published).

Conceptual flow

User row only. No public profile yet.

↓
Company row created 3. Company record (required)

ownerId = user · name = business name or “Trading as [email local part]” if they choose email-only label · slug unique · default visibility: not published.

↓
Stripe 4. Subscription checkout

User chooses a subscription: Partner listing plan (marketplace) or Marketing plan (Google Ads / Google SEO). Uses companyId in metadata. Works even if profile is incomplete or unpublished.

↓

No public URL in search. Dashboard shows “Complete profile to go live” (optional).
